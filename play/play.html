Engine Documentation

Comprehensive details regarding the underlying mechanics, privacy constraints, and terms of service utilized by the BistroX Engine.

1. Data & Privacy Independence

BistroX is architected as a pure, client-side web application. This means the engine operates entirely within the confines of your browser's local memory.

  • No Backend Servers: Your game states, recipe progression, and configuration settings are never transmitted across the network.
  • Cloud Database API: We now utilize secure, per-user Cloud database storage mapping to your unique user ID instead of local browser caching. Your progress follows you on any device.
  • Zero Analytics: There are no tracking scripts, advertising pixels, or telemetry modules embedded in this software.

2. Gameplay Abstraction

BistroX is a logic-puzzle game themed around culinary arts. The mechanics rely on abstract string matching (e.g., combining the string ID onion with the string ID knife via the slice method).

While the theories presented are inspired by real-world cooking logic, they are drastically simplified. Please do not attempt to learn genuine culinary health and safety protocols from this software.

3. Licensing & Fair Use

Access to the BistroX engine is provided free of charge for personal entertainment. The structural code, styling, and specific game logic matrix remain the intellectual property of the developer. Reproduction or commercial resale of the software framework is prohibited without consent.

Asset Pipeline & Sourcing

Creating a beautiful visual experience requires high-fidelity assets. We credit the following sources for making this possible:

  • Photography: High-resolution cinematic background photography is dynamically sourced via the Unsplash API. Specific thanks to independent culinary photographers.
  • Vector Graphics: Game ingredient items (e.g., onions, beef, tools) utilize algorithmically generated SVG data-URIs to maintain zero-latency loading and infinite scaling.
  • Typography: The interface leverages 'Inter' for highly legible system text and 'Jost' for geometric, cinematic headers, provided by Google Fonts.
  • Icons: Menu and action icons are sourced from open-source feather icon libraries.

Future Implementation

Currently, the BistroX engine focuses on visual and logical processing. In future builds, a native Web Audio API wrapper is planned to introduce atmospheric kitchen soundscapes and interactive slicing/searing sound effects.
